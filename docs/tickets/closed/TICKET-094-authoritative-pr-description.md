# TICKET-094: Authoritative PR description generation

Goal: Generate PR description from structured work metadata and actual execution result — not a raw prompt dump.

Difficulty: medium
Risk: low
Recommended backend: codex
Recommended model: gpt-5.4
Suggested MR Title: Generate PR descriptions from structured metadata and execution results

## Problem

Current MR body is a minimal template:
```
## GAH fix mode

Ticket: TICKET-NNN title
Backend/model: `codex` / `gpt-5.4`
Branch: `gah/gah-1783134254`
Target: `main`
Validation passed: true

Generated by `gah dispatch`.
```

This lacks:
- problem statement from the ticket
- what changed (files, diff summary)
- acceptance criteria check
- attempts/fallback information
- known limitations
- why the change exists

Meanwhile, the description is generated AFTER the implementation — the agent's work output and validation results are fresh and available but not included.

## Acceptance Criteria

1. PR description sections (when metadata available):
   - Work Item: ID and title
   - Problem: why the change exists
   - Goal: what was intended
   - Changes: files changed
   - Validation: commands and outcome
   - Backend/model used: actual routed identity
   - Attempts: started/completed, fallback used
   - Failure/stop state: if applicable
   - Branch: source branch
2. Missing sections are omitted, not emitted as empty
3. Raw agent prompt text is never dumped into the description
4. Secrets are never included
5. Backward compatible: descriptions generated without structured metadata fall back to existing template
6. Tests for metadata-rich, metadata-poor, and experiment mode descriptions

## Affected Files

- `src/dispatch.rs` — MR body generation in fix, improve, experiment functions

## Constraints

- Dependencies: TICKET-091, TICKET-092 (work identity and structured metadata)
- Do not change provider code
- Do not change sync or status
- Do not include secrets
- Do not fabricate validation results
